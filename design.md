# イントロクイズ企画 仕様書

## 1. 目的

文化祭企画として、画像を使ったイントロクイズゲームを実施する。管理画面で出題画像や効果条件を操作し、ゲーム画面に表示される画像を見て回答する形式とする。

## 2. システム概要

- フロントエンド: React
- バックエンド: Python
- 画像処理: Python から `ffmpeg` を利用
- 公開方法: 同一PC内で `localhost` 上にゲーム画面と管理画面を公開

本システムは、以下の2画面で構成する。

1. ゲーム画面
2. 管理画面

## 3. 基本ルール

- 5人対戦のポイント制とする
- 基礎ポイントは 1000 点とする
- 効果適用数を `n` とし、獲得点は `1000 × 1.5^n` とする
- `n` は 0〜5 の範囲で扱う
- 画像効果はランダムに複数選択されるが、同一効果の重複は不可とする

## 4. ゲーム進行フロー

### 4.1 基本フロー

1. ルーレット表示
2. 効果個数決定
   - 0〜5 の乱数で決定する
   - ただし 1 に重みを持たせる
   - 重みの値は管理画面で調整可能とする
   - 初期値は設定ファイルから読み込む
3. ルーレット2
   - 適用する効果を一気に決定する
   - 効果の重複はなし
4. 効果適用済みの拡大画像を表示する
5. 指定カウント後に縮小を開始する
   - カウント秒数は管理画面で調整可能とする
   - 初期値は設定ファイルから読み込む
   - 縮小速度も管理画面で調整可能とする
6. 回答待ち状態に移行する
7. 管理画面から停止することで回答受付を行う
   - スペースキーまたはボタンで停止可能
8. 正解処理
   - 正当なら正解者を選択する
   - ポイントを付与する
   - 次の問題に戻る
9. 不正解処理
   - 不正解なら再度スペースを押して再開する

### 4.2 停止と再開

- 縮小中に管理画面から停止できる
- 停止操作後、回答者の回答を受け付ける
- 誤答の場合は同じ問題のまま再開できる

## 5. 画像処理仕様

### 5.1 画像ソース

- サーバーに保存されている画像を対象とする
- 管理画面で出題画像を選択する
- 選択方法は一覧形式を想定する

### 5.2 表示位置

- 管理画面で、画像の拡大中心を指定可能とする
- 指定位置がない場合はランダムな場所を中心にする

### 5.3 効果一覧

ランダム処理対象の効果は以下とする。

- 色反転
- モザイク
- モノクロ
- 上下反転
- 一部隠し

### 5.4 効果選択ルール

- 効果個数 `n` を決定後、候補効果から `n` 個を選ぶ
- 選択済み効果は重複しない
- 選択順は内部的に保持してもよいが、見た目は同時適用でよい

### 5.5 拡大・縮小表示

- 画像は最初に拡大表示する
- 一定カウント後、徐々に縮小する
- 縮小は管理画面で調整する速度に従う

## 6. 画面設計

### 6.1 ゲーム画面

#### レイアウト

- 画面全体を 3 分割した場合、左側 2、右側 1 の比率を基本とする
- 左上に画像を配置する
- 下カラムに 5 人の現在得点を表示する
- 右カラムに以下を表示する
  - 制限時間
  - 現在適用中の画像効果
  - 現在のターン数

#### 表示要素

- 出題画像
- 効果中アイコンまたはテキスト
- 残り時間
- 5人分の得点一覧
- 現在のゲーム状態表示

#### 見た目の方針

- 細かい装飾は実装時に調整可能とする
- 文化祭の会場で見やすいよう、文字は大きめにする
- 重要情報は高コントラストで表示する

### 6.2 管理画面

#### 管理画面レイアウト

- 全体を 2 分割した場合、1:2 の左側 1 の領域を使用する
- 画面の左上半分を占有するように配置する
- 残り領域は自由に使えるが、操作性を優先する

#### 管理機能

- 回答者の点数調整
  - 数字入力で加算・減算を可能にする
- 縮小のストップ
- 縮小速度の調整
- 縮小前カウントの長さ調整
- ルーレットの開始
- ルーレット確率の重さ調整
- 出題画像の選択
- 指定位置の設定、またはランダム選択切り替え
- リザルトボタン（ゲームエンド）
- 初期化ボタン（スコア等の初期化）

#### 操作方法

- ボタン操作を基本とする
- 主要操作はスペースキーでも代替可能とする
- 誤操作防止のため、停止や点数反映には確認を挟んでもよい

## 7. 管理パラメータ

管理画面で調整する値は、設定ファイルのデフォルト値を初期値として読み込む。

### 調整対象

- 効果個数のルーレット重み
- 縮小速度
- 縮小前カウント秒数
- 点数調整値
- 出題画像の選択状態
- 拡大中心の指定方法

### 設定方針

- 起動時に設定ファイルを読み込む
- 管理画面で変更した値は即時反映する
- 必要に応じてローカル保存またはバックエンド保存を行う

## 8. バックエンド仕様

### 8.1 役割

- 画像一覧の提供
- 画像処理の実行
- ゲーム状態の保持
- 点数管理
- 管理画面からの操作反映

### 8.2 画像処理

- Python から `ffmpeg` を呼び出して処理する
- 画像の拡大、縮小、効果適用を制御する
- 処理結果はゲーム画面へ送信する

### 8.3 状態管理

- 現在の出題画像
- 適用効果一覧
- 効果個数 `n`
- 縮小開始タイミング
- 停止状態
- 各プレイヤーの点数

## 9. フロントエンド仕様

### 9.1 React 構成

- ゲーム画面と管理画面を React で実装する
- 画面状態は共通ストアで同期する
- タイマー、ルーレット、停止状態をリアルタイムに反映する

### 9.2 連携

- バックエンドと API で通信する
- 必要に応じて WebSocket を使い、ゲーム状態を即時反映する

## 10. 画面遷移・状態

### 状態一覧

- 待機
- ルーレット表示
- 効果個数決定
- 効果決定
- 画像表示
- カウントダウン
- 縮小中
- 停止中
- 回答受付中
- 正解処理中
- 次問待機

### 遷移条件

- 管理画面の開始操作で待機から進行へ移る
- 縮小前カウント終了で縮小中へ移る
- 停止操作で回答受付中へ移る
- 正誤判定後、次の問題に戻る

## 11. エラー・例外処理

- 画像ファイルが存在しない場合は管理画面で警告を表示する
- `ffmpeg` 実行失敗時は処理を中断し、エラー表示を行う
- ランダム効果が0件となる場合は、そのまま無効果で表示する
- 通信エラー時は再接続を試行し、ゲーム進行を止めない設計とする

## 12. 非機能要件

- 同一PC上で動作し、`localhost` でアクセスできること
- 会場利用を想定し、UI は遠目でも視認しやすいこと
- 操作は最小限の手数で行えること
- 短時間で繰り返し出題できること

## 13. 確定事項（運用）

- 管理画面の認証は不要
- 点数は問題正解時に管理者が正解者を選択して自動付与（`1000 × 1.5^n`）
- 画像一覧の取得元は `backend/images/`
- 効果個数の重みは 0〜5 それぞれ整数で管理画面から調整可能
- 縮小を停止したら管理画面に「正解判定」ボタン（プレイヤー選択）を表示する

## 14. 補足

本仕様書は、文化祭当日の運用を前提にした実装指針である。必要に応じて、操作性を優先して細部を調整する。
